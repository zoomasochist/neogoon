Neogoon

Gooning encouragement utility. Latest release.

Introduction

Neogoon is a gooning encouragement utility functionally inspired by Edgeware, in turn inspired by Elsavirus. Like Edgeware, it is open source: https://github.com/zoomasochist/neogoon.

Neogoon was founded out of frustration with Edgeware's poor maintenance record and user experience, so while Neogoon "does the same thing" as Edgeware, it sets out to do it better in every possible way.

Neogoon is still a work-in-progress. Some features may not work properly, and some may not be implemented at all. But I hope it currently and continues to serve you well.

Comparison

Neogoon

  • Actively developed

  • Single dependency-free binary

  • Works on macOS, Linux, and Windows

  • Configured through a text file (web configurator WIP)

  • Some things don't work yet

  • Incompatible package format

Edgeware

  • More mature

  • More packages

  • Configured through a GUI

  • Sees little maintenance

  • Has to be installed

  • Only works on Windows

  • Requires Python

  • Poor UX

Documentation

neogoon.toml

neogoon.toml is where the Neogoon client is configured, rather than from a graphical interface à la Edgeware.

The example neogoon.toml is very well documented, so its efforts won't be duplicated here.

Sets

Sets are what Neogoon calls packages.

Sets, as they are loaded into Neogoon, are zip files containing directories for assorted media, and a set.toml at the root to configure the set and provide textual data.

In Neogoon, all directories used for storing media (e.g. images) can contain further images to compartmentalise their content. Nested folders will be followed and additionally searched for relevant media.

A simple uncompressed set would look something like this: 


    \---MySet
    |   set.toml
    |
    \---images
            image.png

File: set.toml

set.toml is a TOML file that configures the set with textual content, and a few settings.

The following entries are expected:

urls

An array of string URLs that may be opened as part of the open-websites annoyance.

texts

An array of strings used in a variety of places — they're overlayed over popups and sent as notifications, for example. You should probably mix a collection of short and long texts, and shouldn't be written in first person. Use prompts for that.

prompts

Prompts are like texts, but used specifically for text (usually) written in first person. Prompts will be presented along-side a text box, and the gooner will have to repeat the text in the text box to close it. They should probably be demeaning.

filenames

An array of strings used with the drive-filler annoyance. When enabled, these filenames will be chosen from as the name of images that are saved.

Directory: images

The images directory contains images used for Neogoon features like popups and wallpaper setting. PNGs and JPGs are officially supported.

Directory: animated

The animated directory contains animated (but not video) media used for popups. The only supported format will be GIF, but there isn't support for animated popups yet.

Directory: videos

The videos directory contains videos used for popups. There isn't support for videos popups yet, so the list supported formats is indeterminate.

Getting started

Configure your neogoon.toml

Grab the example neogoon.toml (click "Raw", right click and "Save Page As...") and save it somewhere sensible. The example neogoon.toml is also used for testing Neogoon, so it probably won't exactly suit your needs — it's best to thoroughly read it and configure it to your liking before continuing. Every configurable part of Neogoon should make sense, so if something doesn't, please report it!

Grab a set

Download Neogoon

Download the latest release of Neogoon for your operating system and run it. You should be presented with a popup telling you it's ready. From there, you can access Neogoon from your system tray. Click "Load config" and "Load set" to load your configuration and set respectively, then click "Start".

Note that some anti-virus software may purport Neogoon is malware (understandably). There's not a lot I can do about that. Sorry. Just ignore it! :)

Developers

Hacking on (and hopefully improving) Neogoon is encouraged!

Building

Neogoon is written in Go, so building is very easy. On Linux and macOS, it should only require: 


                $ git clone https://github.com/zoomasochist/neogoon
                $ cd neogoon
                $ go build
                $ ./neogoon
On Windows the process is very similar, but you may have to install GCC (for CGO). The easiest way to get GCC working on Windows is https://jmeubank.github.io/tdm-gcc/.
